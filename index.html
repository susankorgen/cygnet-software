Towers of Hanoi

GitHub

▼ ▲

Animate the classic recursion problem using pure, raw JavaScript, HTML, CSS, and SVG.

The SVG graphics are not downloaded images; they are dynamically generated as SVG XML content when this code runs.

Computed data for each tower height the user chooses (5, 6, ... , 12, 13) is retained during the session in case of a repeat demo.

The code showcases UX and UI development best practices that are independent of coding language and technology stack.

A solid first implementation.

Graphically illustrates the cost of order 2n computation.

Supports a user language choice of English or French and easy translation for other user languages.

Comments at the end of hanoi/main.js list the specific planned updates: UI/UX coding best practices that I have not added to the example yet.

8 days.

A return to JavaScript and front end development after 4 years using a proprietary, object-oriented database language in back end development. Decided to also use all new tools (Atom, GitHub, GitHub desktop) as a change from the proprietary IDE.

8 days for this example began with 2 days of JavaScript: The Good Parts by Douglas Crockford. Page 34 mentions the hanoi problem and provides a 3-line loop.

Recursion can be challenging to visualize. I was inspired to build my hanoi animation as my first JavaScript code example.

As I built the example, I saw I could shape it to demonstrate many UI/UX best practices that I learned during 7 years of front end development using JavaScript, prior to my more recent server side experience.

The past work included building JavaScript UIs and more than one UI generation framework that output HTML and JavaScript
